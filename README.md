# Pre-train Advisor для преобразований отклика

Инструмент для регрессионных задач: до начала обучения модели даёт обоснованную рекомендацию, нужно ли преобразовывать переменную-отклик и какое именно преобразование выбрать, с прогнозом ΔRMSE и доверительным интервалом.

## О проекте

Система анализирует пользовательский датасет, диагностирует распределение отклика Y (асимметрия γ₁, эксцесс, оценка Box-Cox λ̂ с 95% ДИ, тесты на нормальность), и на основе базы знаний из 27 датасетов и 10 моделей выдаёт ранжированные рекомендации по семи преобразованиям. Применимость каждой рекомендации проверяется четырьмя методологическими правилами, выведенными из эмпирического бенчмарка. Дополнительно — режим empirical audit, который обучает все модели × применимые преобразования на пользовательских данных и определяет фактического победителя по RMSE.

### Возможности

- Диагностика отклика Y (γ₁, γ₂, Box-Cox λ̂ с ДИ, Shapiro-Wilk, LRT)
- Top-K рекомендаций преобразований по базе знаний с медианой ΔRMSE и интервалом [P10, P90]
- Четыре формальных правила применимости (Rule-1..Rule-4) с автоматической блокировкой запрещённых преобразований
- Empirical audit: обучение всех моделей × преобразований на пользовательских данных
- Полное ранжирование (model × transform) по фактическому RMSE на hold-out
- Тест Дибольда–Мариано на значимость различий, paired t-test, Cohen's d
- Сверка эмпирических результатов с прогнозом базы знаний
- Веб-интерфейс с визуализацией диагностики, рекомендаций и сравнения стратегий
- Markdown-отчёт с диагностикой и рекомендациями для вставки в работу

## Установка

### 1. Клонирование репозитория

git clone https://github.com/Penek12/Pre-train-Advisor-.git

cd pre-train-advisor

### 2. Установка Python и создание виртуального окружения

python -m venv venv

Активация (Mac/Linux):

source venv/bin/activate

Активация (Windows):

venv\Scripts\activate

### 3. Установка зависимостей

pip install --upgrade pip

Для полного режима с обучением моделей (рекомендуется для локального запуска):

pip install -r requirements-full.txt

Для облачного режима без обучения (только диагностика и рекомендации по KB, ~150 МБ вместо ~1 ГБ):

pip install -r requirements.txt

### 4. Подготовка базы знаний

База знаний `full_results_v5.csv` уже включена в репозиторий в папке `results/`. Если вы хотите перегенерировать её на собственном наборе датасетов:

1. Подготовьте список датасетов в формате (X, y) — внутри скрипта `chapter2_experiments_v6.py`
2. Запустите полный бенчмарк:

python chapter2_experiments_v6.py

3. Результаты сохранятся в `results/full_results_v5.csv` и будут автоматически подхвачены advisor-ом при следующем запуске

Прогон занимает 30–90 минут в зависимости от мощности машины и числа датасетов.

## Использование

1. В боковой панели загрузите CSV, TSV или Excel-файл с числовым откликом и фичами (или включите «взять демо-датасет» для пробного прогона)
2. При необходимости загрузите свою базу знаний — иначе будет использована стандартная из `results/full_results_v5.csv`
3. Выберите колонку Y, колонки фичей X и модель из выпадающего списка
4. Установите количество рекомендаций (top-K)
5. Ознакомьтесь с результатами по четырём вкладкам:
   - **Диагностика** — γ₁, γ₂, Box-Cox λ̂, LRT-тесты, гистограмма Y и QQ-сравнение
   - **Рекомендации** — top-K преобразований с медианой ΔRMSE, интервалом [P10, P90], вероятностью улучшения и обоснованием
   - **Сравнение стратегий** — empirical audit всех моделей × применимых преобразований, победитель с фактическим RMSE и DM-тестом
   - **Отчёт** — скачиваемый markdown с диагностикой и рекомендациями

## Случай использования №1: pre-train advisor

Этот сценарий — для ML-инженера, который только что получил новый датасет и хочет за минуту понять, нужно ли преобразовывать отклик. Никакие модели не обучаются, вся логика — поиск по базе знаний.

После загрузки CSV в боковой панели на вкладке **Диагностика** инструмент посчитает асимметрию γ₁, эксцесс, оценку Box-Cox λ̂ с 95% ДИ, проведёт тест Шапиро–Уилка и LRT-тесты на частные случаи λ = 0 (логарифм) и λ = 1 (без преобразования). γ₁ = +1.6 интерпретируется как «выраженный правый хвост», λ̂ ≈ 0 — «оптимум близок к логарифму», Shapiro p < 0.001 — «нормальность отвергнута». Внизу вкладки — гистограмма Y и QQ-plot для визуальной проверки.

На вкладке **Рекомендации** алгоритм работает процедурно: берёт γ₁-бин пользовательских данных, фильтрует базу знаний по этому бину и выбранной модели, ранжирует преобразования по медиане ΔRMSE. Каждая рекомендация показана с порядковым номером, классом преобразования, четырьмя ключевыми цифрами (медиана ΔRMSE, интервал P10–P90, вероятность улучшения, вероятность значимого улучшения) и текстовым обоснованием. Применимость проверяется четырьмя правилами: для древесных моделей все преобразования блокируются Rule-1, для Y ∈ [0, 1] — log и Box-Cox блокируются Rule-3, для γ₁ ≥ 3 — Box-Cox и Yeo-Johnson блокируются Rule-4.

В expander «Сырые правила KB» показана полная таблица всех правил для γ₁-бина пользователя — это для прозрачности: видно, из каких именно строк бенчмарка выведены рекомендации. На вкладке **Отчёт** можно скачать markdown с полной диагностикой и рекомендациями для вставки в дипломную работу или передачи коллегам.

Время выполнения: 5–15 секунд. Подходит для облачного деплоя на Streamlit Community Cloud.

## Случай использования №2: полноценный empirical audit

Этот сценарий нужен, когда пользователь хочет не просто получить рекомендацию, а подтвердить её обучением на собственных данных. Требует полного набора зависимостей (`requirements-full.txt`) и chapter2_experiments_v6.

На вкладке **Сравнение стратегий** по умолчанию доступны пять моделей: МНК, Ridge, Lasso, Random Forest, XGBoost. Можно расширить до десяти, добавив MLP, Gamma GLM, Tweedie GLM, XGB-Gamma и XGB-Tweedie. Каждая модель тестируется с применимыми преобразованиями — для линейных все семь, для остальных только baseline.

После нажатия кнопки **Запустить** начинается обучение с progress bar. Для типового датасета на 1000 строк × 10 фич с пятью моделями это занимает 30–60 секунд. Внутри для каждой пары (модель, преобразование) запускается 5-fold cross-validation, затем модель переобучается на полном train и оценивается на hold-out. Для пар с преобразованием Y обратное преобразование делается с поправкой Дуана, и RMSE считается уже в исходном пространстве. Относительно baseline считается тест Дибольда–Мариано.

Результат состоит из трёх частей. **Карточка победителя** наверху — комбинация (модель, преобразование) с минимальным RMSE на test, например «Lasso + Yeo-Johnson, RMSE = 4.52, ΔRMSE = −0.5%, DM p-value = 0.034». **Полное ранжирование** — таблица всех пар, отсортированная по RMSE; видно, есть ли альтернативы с близким RMSE, но более простой моделью. **Гистограмма «Лучшее преобразование для каждой модели»** показывает чувствительность моделей к выбору преобразования.

В expander «Сверка: эмпирика vs прогноз KB» сопоставлены прогнозы из базы знаний и фактические результаты. Большие расхождения сигнализируют, что обобщённые правила KB плохо переносятся на конкретный датасет.

Этот режим используется когда: высокие требования к точности; датасет сильно отличается от тех, что в бенчмарке (γ₁ > 3, ограниченный Y, малая выборка); нужно отчитаться по фактическим экспериментам.

## Тестовые данные

Для тестирования можно использовать датасеты из репозитория UCI ML:

- California Housing — цены на недвижимость (γ₁ ≈ 1.0, идеальный кейс для бина L–M)
- Diamonds — цены на бриллианты (γ₁ ≈ 1.5, кейс для бина M–H)
- Forest Fires — площадь выгорания (γ₁ ≈ 12.8, экстремальный случай для тестирования Rule-4)
- Communities Crime — индекс преступности в [0, 1] (тестирование Rule-3 для ограниченного Y)

Все они также представлены в базе знаний `results/full_results_v5.csv`.

## Тестирование правил

Модуль `transform_applicability_rules.py` содержит самопроверку на семи кейсах из бенчмарка:

python transform_applicability_rules.py

Тест прогоняет четыре правила на четырёх «дефектных» датасетах (Communities_Crime, CPU_Performance, RAND_HIE, Forest_Fires) и трёх контрольных (Auto_MPG, WineQuality_Red, California). Все семь тестов должны пройти.
